    Dosya: app.json
    Mevcut: "bundleIdentifier": "com.signalvsnoise"
    Yapılacak: "com.corut.glint" gibi gerçek, benzersiz bir identifier yaz.
    Android package da aynı şekilde değişmeli (mevcut: "com.signalvsnoise").

    NOT: Diğer tüm maddeler tamamlandıktan sonra en son adım olarak yapılacak.
    Önce Apple Developer Portal'da Certificates, Identifiers & Profiles →
    Identifiers → "+" ile yeni Bundle ID oluştur, sonra app.json'a yaz.

1.2 Scheme Değiştir
    Dosya: app.json
    Mevcut: "scheme": "signalvsnoise"
    Yapılacak: "scheme": "glint"
    Deep link'ler bu scheme üzerinden çalışır, uygulama adıyla tutarlı olmalı.

    Uygulama lokal veri tutuyor (AsyncStorage ile skor, ayarlar, istatistikler).

    App Store Connect'e eklemelisin.
    Notion, GitHub Pages veya kendi siten üzerinden yayınlayabilirsin.
    KVKK, GDPR, CCPA uyumlu olması stratejik avantaj sağlar.

    - Uygulama hiçbir kişisel veri toplamıyor

    Ocak 2026'da yeni yaş sistemi geldi: 4+, 9+, 13+, 16+, 18+

[2] ÖNEMLİ - YAPILMAZSA RED RİSKİ VAR
--------------------------------------------------------------

2.1 Kullanılmayan Bağımlılıkları Temizle
    Şu paketler yüklü ama kullanılmıyor olabilir:
    - expo-web-browser (plugins'te var ama kodda kullanılıyor mu kontrol et)
    - react-native-svg (kullanılıyor mu kontrol et)
    - @expo-google-fonts/inter (font yüklenmiş mi kontrol et)
    Apple gereksiz izin isteyen veya şişirilmiş uygulamaları reddedebiliyor.
    Kullanılmayanları kaldır: npm uninstall <paket-adı>
    app.json plugins'ten de expo-web-browser'ı kaldır.

2.2 EAS Build Yapılandırması Oluştur
    eas.json dosyası yok. App Store'a göndermek için gerekli.
    Komut: npx eas-cli init && npx eas build:configure
    Ardından eas.json'da production profili ayarlanmalı:
    {
      "build": {
        "production": {
          "ios": { "buildNumber": "1" }
        }
      }
    }

2.3 Nisan 2026 SDK Gerekliliği
    28 Nisan 2026'dan itibaren uygulamalar iOS 26 SDK ile build edilmeli.
    Şu an Expo SDK 54 kullanılıyor (React Native 0.81.5).
    Expo'nun iOS 26 SDK destekleyen sürümüne geçiş gerekecek.
    Expo blog ve changelog takip edilmeli.

2.4 Uygulama Stabilitesi
    - Uygulama herhangi bir anda çöküyorsa reddedilir.
    - Farklı iPhone modellerinde ekran bozulması, uyumsuz UI reddedilme sebebi.
    - Arka plana atıp geri gelme senaryosu düzgün çalışmalı.
    - Düşük bellek durumunda crash olmamalı.
    - Bilinen sorun: react-native-worklets versiyon uyumsuzluğu
      (JS 0.7.4 vs native 0.5.1) — production build'de sorun çıkarabilir.

2.5 Splash Screen Güncelle
    Dosya: app.json → splash
    Mevcut: backgroundColor "#FFFFFF"
    Dark mode'da beyaz splash → siyah uygulama geçişi kötü görünür.
    userInterfaceStyle "automatic" yapılabilir veya
    koyu arka planlı splash ekranı da hazırlanmalı.

[12] UYGULAMA CİLASI - DAHA ŞIK VE PROFESYONEL İÇİN
--------------------------------------------------------------

Aşağıdakiler App Store için zorunlu değil ama uygulamanın "indie" den
"premium" hissine geçmesini sağlayacak dokunuşlar. Öncelik sırasına göre:

--- A. ANİMASYON & HİSSİYAT (En Çok Fark Yaratan) ---

12.1 Ekranlar Arası Geçiş Animasyonları
     Şu an menü → oyun → ayarlar geçişleri keskin (conditional render).
     Fade + slide animasyon eklenirse "cilalı uygulama" hissi verir.
     React Native Reanimated layout animations veya
     shared element transitions kullanılabilir.
     Zorluk: Orta | Etki: Yüksek

12.2 Doğru Tap'te Parçacık (Spark) Efekti
     Doğru kelimeye dokunduğunda yeşil mini konfeti/spark patlaması.
     5-8 adet küçük daire dışa doğru yayılıp kaybolur.
     Reanimated ile lightweight implementasyon yapılabilir.
     Zorluk: Orta | Etki: Yüksek

12.3 Game Over Screen Shake
     Öldüğünde tüm ekranın 300ms hafif sarsılması.
     Reanimated withSequence ile basit implementasyon.
     Dramatik etki, kullanıcı "vay" der.
     Zorluk: Kolay | Etki: Yüksek

12.4 Kelime Kartlarına Giriş Animasyonu
     Kelimeler şu an yukarıdan doğrudan düşüyor.
     Spawn olurken küçük bir scale-in (0.8 → 1.0) + fade-in eklenirse
     daha organik görünür.
     Zorluk: Kolay | Etki: Orta

12.5 Buton Basma Haptik Geri Bildirimi
     Ana menü butonlarına basınca hafif titreşim eklenebilir.
     Şu an sadece oyun içi tap'lerde haptic var.
     Zorluk: Kolay | Etki: Düşük-Orta

--- B. GÖRSEL TASARIM (Profesyonel İzlenim) ---

12.6 Custom Font (Tipografi)
     Sistem fontu yerine özel font kullanmak uygulamayı anında
     "tasarlanmış" gösterir. Öneriler:
     - Başlık: Space Grotesk veya Outfit (Bold/Black)
     - Gövde: Inter veya DM Sans
     @expo-google-fonts zaten yüklü, Inter mevcut.
     Zorluk: Kolay | Etki: Yüksek

12.7 Kelime Kartı Tasarım Varyasyonları
     Şu an tüm kartlar aynı görünüyor (beyaz kutu).
     Signal ve Noise kartların arasında görsel ipucu OLMAMALI (oyunun amacı bu)

     - Hafif gradient arka plan
     - İnce iç gölge (inner shadow hissi)
     - Mikro doku (subtle noise texture)
     Zorluk: Kolay | Etki: Orta

12.8 Ana Menüde Arka Plan Animasyonu
     Floating dot'lar statik duruyor. Bunlar yavaşça hareket etse
     (drift animasyonu, 15-20 saniyelik döngü) menü daha canlı olur.
     Zorluk: Kolay | Etki: Orta

12.9 Oyun Alanı Arka Plan Deseni
     Oyun alanı şu an düz renk. Çok hafif, neredeyse görünmez
     bir grid/dot pattern eklenirse derinlik hissi verir.
     Zorluk: Kolay | Etki: Düşük-Orta

     - Kategoriler: hayvanlar, eşyalar, meyveler/sebzeler, doğa,
       renkler, meslekler, vücut, genel kavramlar

12.6 Zorluk Sistemi
     - Logaritmik zorluk eğrisi (asimptotik, asla minimuma ulaşmaz)
     - Fall: 4200ms → ~1000ms, Spawn: 2200ms → ~400ms, Noise: %28 → ~%75
     - 10 farklı renkte progress bar döngüsü

12.7 Tema Sistemi
     - Light ve Dark mode (sistem temasına otomatik uyum)
     - ThemeContext + useColors hook
     - Tüm komponentler dinamik tema destekli

12.8 İstatistikler
     - Toplam oyun, toplam skor, en iyi skor, en iyi kombo
     - AsyncStorage'da kalıcı
     - Ana menüde ve game over'da gösterim

12.9 Ayarlar Ekranı
     - Haptic feedback açma/kapama
     - En iyi skoru sıfırlama
